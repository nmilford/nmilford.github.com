Installing Apache Cassandra on CentOS

Jun 19th, 2010

From Cassandra’s site:

The Apache Cassandra Project develops a highly scalable second-generation distributed database, bringing together Dynamo’s fully distributed design and Bigtable’s ColumnFamily-based data model.

It’s one of the more popular NoSQL data stores out there and at Outbrain we’ve been moving some parts of our service from MySQL to it.

I love it as an Ops guy because it just works… I set it up, fire it up and it goes. Mind that you need to set it up right in the beginning, but that is another thing all together and I won’t get into configuration and implementation, just deployment.

It is best if you have JDK 6 Update 20+ installed.

Make important directories and files.

I like to set it up so that the logs, commitlog and data are living in directories a tier below there the tarball gets unpacked so I can simply unpack the next version and fire it up to upgrade.

Lets get Cassandra.

Put it in the PATH.

Link the conf directory to /etc.

At this point I’d tweak /etc/cassandra/storage-conf.xml and set my seeds so nodes know where to start gossping to discover each other.

Outbrain is lucky to have Ran Tavory in our midst who is the creator of the Hector java client for Cassandra as well as contirbutor to the Cassandra project in general. He suggested the inclusion of mx4j for taking a peek at the JMX counters that Cassandra reports. It’s pretty nice for the developers and makes it easier to find what you need for adding JMX counters to your monitoring suite (we use OpenNMS)

Add mx4j.

My first ~nix was OpenBSD so I have some paranoia in me and I hate running daemons as root. I also like to create a seperate user for developers to log in with and tweak the server while we’re still learning the service.
